telegraph-Image：搭建你的专属开源图床

前言

在本教程中，我们将搭建一个图床项目，使用 telegraph-Image 来上传图像，同时将其部署在 Cloudflare Pages 上，并借助 D1 数据库来存储图像信息。该图床服务便于与 Hexo 博客结合，方便博主上传、管理图片。

优点

  1. 部署简单：Cloudflare Pages 提供一键部署功能，无需服务器管理。
  2. 全球加速：借助 Cloudflare 的 CDN，使图像加载速度快且稳定。
  3. 低成本或免费：D1 数据库和 Pages 部署计划对小项目友好，几乎无成本。
  4. 开源且可扩展：Telegraph-Image 开源，支持自定义功能，便于个性化。
  5. 与 Hexo 兼容：可与 Hexo 博客系统集成，方便图片管理。

项目地址：

https://a9f2e50e.telegraph-image-v8j.pages.dev/

开始部署

1. Pages 部署 telegraph-Image 项目

访问 telegraph-Image 先给作者点击 Star 后再点击 Fork

  • 回到 Cloudflare 打开左侧 Workers 和 Pages > 概述 > 创建 > Pages > 连接到 Git > 连接到 GitHub > 选择 telegraph-Image 项目最后保存并部署即可

  • 打开 telegraph-Image 设置 > 左侧 Tab 栏点击运行时 > 放置 > 智能 > 保存

  • 如果您有自定义域名，可以在 Cloudflare 设置中添加，并在域名设置中指向 Cloudflare 提供的 CNAME 地。

  • 本文使用 Cloudflare 默认分配的域名 不做自定义域名内容。

2. 创建管理后台

  • 打开左侧 Workers 和 Pages > D1 SQL 数据库 > 创建 > 数据库名称img(可以是自定义) > 创建

  • 进入IMG数据库 > 控制台 > 粘贴以下代码 > 点击执行 > 提示此查询已成功执行即可。

  • 打开 telegraph-Image 设置 > 左侧 Tab 栏点击绑定 > D1 数据库 > 变量名称IMG > D1 数据库选择img > 保存

  • 打开 telegraph-Image 设置 > 左侧 Tab 栏点击变量和机密 > 添加如下内容

    • 变量名称 BASIC_USER 值为你的后台管理员用户名-
    • 变量名称 BASIC_PASS 值为你的后台管理员密码-
    • 保存

  • 回到 telegraph-image 部署 > 所有部署 >找到最新部署那一条点击三个点重新部署等待片刻即可

  • 目前 TG 接口已失效 我使用的是vviptuangou和58img 实测可以成功上传接口

总结

通过使用 telegraph-Image、Cloudflare Pages 和 D1 数据库，我们可以快速搭建一个开源的图床项目，并与 Hexo 博客无缝集成。这个图床不仅可以提供稳定的图像存储，还能利用 Cloudflare 的全球加速和高可用性服务，方便进行大规模的图片管理和展示。
